Simulation Event Calculations

This design document provides a detailed breakdown of how player attributes shape match events in the Variant B match simulation engine. It details the mathematical formulas, attribute weights, and display statuses of stats involved in each gameplay check.

Important Gameplay Rule: Skill Compression
During match resolution, all player attributes accessed via getPlayerStat()—except overall, endurance, and injuryRate—are dynamically compressed toward a baseline of 70: effective_value = 70 + (raw_value - 70) * skillCompressionFactor; // Default factor is 0.60 This compresses extreme attribute spreads to preserve parity while maintaining individual profiles. The calculations listed below operate on these effective values.

1. Carry (Dribbling / Ball Progression)

Triggered when a player moves forward carrying the ball. Success keeps possession and advances the attacking depth; failure triggers a turnover winner check.

Attributes used:
speedShown
flairShown
// src/app/services/match.simulation.variant-b.service.ts successChance = baseCarrySuccessChance; // Derived from carry type and defensive pressure successChance += (getPlayerStat(carrier, 'speed') - 70) * 0.002; successChance += (getPlayerStat(carrier, 'flair') - 70) * 0.0015; // Clamped to prevent complete certainty finalChance = clamp(successChance, 0.30, 0.86);

2. Pass Resolution

Determines the success rate when an attacker attempts to pass the ball. If a pass fails, the defender list is evaluated for interception.

Attributes used:
shortPassingShown
longPassingShown

// src/app/services/match.simulation.variant-b.service.ts // 1. Base pass quality rating baseQuality = (getPlayerStat(passer, 'shortPassing') + getPlayerStat(passer, 'longPassing')) / 2 + 16.0; // 2. Pass success chance calculation (subject to defensive pressure and style modifiers) passSuccessChance = baseQuality * pressureModifier * styleModifier; finalChance = clamp(passSuccessChance, 0.20, 0.95); // 3. Vision check (used to calculate pass-intercept safety/risk) visionScore = getPlayerStat(passer, 'vision');

When a carry or pass fails, the simulation determines which defending player should be credited with the tackle or interception. Each candidate defender receives an attribution score based on proximity and key attributes.

4. Open Play Shooting

Shots from open play pass through two distinct gates: hitting the target, and beating the goalkeeper.

Attributes used:
shootingShown
handlingShown
reflexesShown
// src/app/services/match.simulation.variant-b.service.ts // 1. Hitting the Target Gate onTargetChance = baseChance + (getPlayerStat(shooter, 'shooting') - 70) * activeTuning.onTargetSkillScale; // 2. Beating the Goalkeeper Gate (if Shot is On-Target) keeperSkill = (getPlayerStat(gk, 'handling') + getPlayerStat(gk, 'reflexes')) / 2; goalChance = baseChance + (getPlayerStat(shooter, 'shooting') - keeperSkill) * activeTuning.goalChanceSkillScale;

7. Injury Rolls

Evaluates active players after certain events (fouls, tackles, etc.) to check if they incur an injury.

Attributes used:
injuryRateHidden
// src/app/services/match.simulation.variant-b.service.ts // injuryRate is NOT compressed during this check injuryRate = attrs.injuryRate.value; finalChance = baseChance * (injuryRate / 50) * siteModifier; // baseChance: ~0.0104 (contact) / ~0.0017 (non-contact) // siteModifier: 1.5x (foul victim), 1.25x (tackle participant), 1.0x (others)
